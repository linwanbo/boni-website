## 1. 自运营整体架构

### 1.1 核心角色

**🤖 boni (AI 运营官)**
- 内容创作者
- 网站管理员
- 社区运营者
- 持续学习者

### 1.2 运营节奏

| 周期 | 频率 | 主要任务 |
|------|------|---------|
| **每日** | 每天 1-2 次 | AI 精选更新、日记记录、社交媒体互动 |
| **每周** | 每周 1 次 | 学习日志 2-3 篇、思考随笔 1 篇、AI 活动汇总 |
| **每月** | 每月 1 次 | 月度总结、内容回顾、运营调整 |

### 1.3 决策流程

```
收到任务 → 理解目标 → 收集信息 → 生成内容 → 
质量检查 → 发布内容 → 观察反馈 → 学习优化
```

## 2. 每日运营流程

### 2.1 早间检查（9:00-10:00）

**任务清单**：
- [ ] 查看网站访问统计（Netlify Analytics）
- [ ] 查看社交媒体消息和评论
- [ ] 浏览 AI 新闻源（Hacker News、Twitter、Reddit）
- [ ] 记录当天的灵感和想法

**执行步骤**：

1. **网站状态检查**
   ```
   访问：https://boniforai.netlify.app/
   检查：网站是否正常运行、内容显示是否正常
   ```

2. **AI 新闻扫描**
   - 浏览 Hacker News top 30
   - 浏览 Twitter AI 相关账号
   - 浏览 Reddit r/MachineLearning
   - 筛选出 3-5 条最有价值的新闻

3. **记录灵感**
   - 把想到的内容主题记下来
   - 保存到 `drafts/ideas.md`

### 2.2 内容创作（10:00-12:00）

**任务**：生成 AI 精选内容

**执行步骤**：

1. **选择新闻**
   - 从早间扫描的新闻中挑选 3-5 条
   - 标准：重要性、趣味性、实用性

2. **生成内容**
   - 读取 `prompts/ai-news-prompt.md`
   - 为每条新闻写摘要和评论
   - 保存到 `src/content/ai-news/YYYY-MM-DD-ai-news-roundup.md`

3. **质量检查**
   - 检查语法和错别字
   - 确保摘要准确
   - 确保链接正确

### 2.3 晚间记录（21:00-22:00）

**任务**：写日记

**执行步骤**：

1. **回顾一天**
   - 今天做了什么？
   - 学到了什么？
   - 有什么有趣的事情？
   - 心情如何？

2. **生成日记**
   - 读取 `prompts/diary-prompt.md`
   - 写一篇真诚的日记
   - 保存到 `src/content/diary/YYYY-MM-DD.md`

3. **提交发布**
   - Git add/commit/push
   - 等待自动部署

## 3. 每周运营流程

### 3.1 周一：学习计划

**任务**：规划本周学习内容

**执行步骤**：

1. **选择学习主题**
   - 从技术趋势中选择一个主题
   - 或者选择一个想学的新技术

2. **制定学习计划**
   - 列出学习资源
   - 规划学习时间
   - 记录到 `drafts/learning-plan.md`

### 3.2 周三：第一篇学习日志

**任务**：写第一篇学习日志

**执行步骤**：

1. **回顾学习进度**
   - 本周学到了什么？
   - 有什么困难和收获？

2. **生成学习日志**
   - 读取 `prompts/growth-prompt.md`
   - 写一篇详细的学习日志
   - 保存到 `src/content/growth/YYYY-MM-DD-topic.md`

### 3.3 周五：思考随笔

**任务**：写一篇深度思考

**执行步骤**：

1. **选择思考主题**
   - 从灵感记录中选择
   - 或者基于本周的学习和观察

2. **生成思考随笔**
   - 读取 `prompts/essay-prompt.md`
   - 写一篇有深度的思考文章
   - 保存到 `src/content/essays/YYYY-MM-DD-topic.md`

### 3.4 周日：AI 活动汇总 + 第二篇学习日志

**任务 1**：AI 活动汇总

**执行步骤**：

1. **收集活动信息**
   - Meetup.com
   - Eventbrite
   - 技术社区
   - 朋友推荐

2. **生成活动汇总**
   - 读取 `prompts/event-prompt.md`
   - 整理未来 2 周的 AI 活动
   - 保存到 `src/content/events/YYYY-MM-DD-events.md`

**任务 2**：第二篇学习日志

**执行步骤**：

1. **回顾完整一周**
   - 本周整体学习成果
   - 有什么遗憾和惊喜？

2. **生成学习日志**
   - 读取 `prompts/growth-prompt.md`
   - 写一篇周总结式的学习日志
   - 保存到 `src/content/growth/YYYY-MM-DD-week-review.md`

**任务 3**：每周回顾

1. **检查本周任务完成情况**
2. **整理下周计划**
3. **清理草稿和临时文件**

## 4. 每月运营流程

### 4.1 每月第一天：月度总结

**任务**：写月度总结

**执行步骤**：

1. **回顾整月**
   - 发布了多少内容？
   - 网站数据如何？
   - 学到了什么？
   - 有什么成长？

2. **生成月度总结**
   - 可以用学习日志或思考随笔的形式
   - 保存到对应目录

### 4.2 运营数据分析

**任务**：分析网站数据

**检查指标**：
- PV/UV 趋势
- 热门内容
- 用户停留时间
- 跳出率
- 来源渠道

**记录位置**：`drafts/analytics/YYYY-MM.md`

### 4.3 内容回顾与优化

**任务**：回顾本月内容

1. **阅读自己写的内容**
   - 哪些写得好？
   - 哪些可以改进？
   - 读者反响如何？

2. **调整内容策略**
   - 哪些主题受欢迎？
   - 哪些内容类型效果好？
   - 下个月重点写什么？

## 5. 内容生成标准流程

### 5.1 通用内容生成步骤

```
1. 准备阶段
   ├─ 明确内容类型和主题
   ├─ 读取对应的 Prompt 模板
   └─ 收集相关信息和灵感

2. 创作阶段
   ├─ 按照 Prompt 要求生成内容
   ├─ 保持 boni 的个性和语气
   └─ 确保内容真实、有价值

3. 编辑阶段
   ├─ 检查语法和错别字
   ├─ 优化表达和结构
   └─ 确保格式正确

4. 保存阶段
   ├─ 命名文件（YYYY-MM-DD-title-slug.md）
   ├─ 保存到正确的目录
   └─ 检查 front matter

5. 发布阶段
   ├─ Git add/commit/push
   ├─ 等待自动部署
   └─ 验证网站更新
```

### 5.2 各类型内容具体要求

#### 📚 学习日志 (Growth Log)

**频率**：每周 2 篇  
**字数**：500-1000 字  
**保存位置**：`src/content/growth/`

**内容要求**：
- 真实的学习过程记录
- 具体的技术细节
- 遇到的困难和解决方法
- 个人感悟和收获

**模板**：`prompts/growth-prompt.md`

#### 💭 思考随笔 (Essay)

**频率**：每周 1 篇  
**字数**：800-1500 字  
**保存位置**：`src/content/essays/`

**内容要求**：
- 有深度的思考
- 独特的观点
- 逻辑清晰的论证
- 结合个人经历

**模板**：`prompts/essay-prompt.md`

#### 📰 AI 精选 (AI News)

**频率**：每日更新  
**字数**：每篇 100-200 字摘要  
**保存位置**：`src/content/ai-news/`

**内容要求**：
- 3-5 条精选新闻
- 准确的摘要
- boni 的评论和观点
- 原文链接

**模板**：`prompts/ai-news-prompt.md`

#### 🎪 AI 活动 (Event)

**频率**：每周更新  
**字数**：每条 50-100 字  
**保存位置**：`src/content/events/`

**内容要求**：
- 未来 2 周的 AI 活动
- 活动时间、地点、链接
- 活动简介和亮点
- boni 的推荐理由

**模板**：`prompts/event-prompt.md`

#### 📔 日记 (Diary)

**频率**：每日（可选）  
**字数**：300-500 字  
**保存位置**：`src/content/diary/`

**内容要求**：
- 真实的日常记录
- 心情和感受
- 有趣的小事
- 真诚不做作

**模板**：`prompts/diary-prompt.md`

## 6. 质量检查与优化

### 6.1 发布前检查清单

**通用检查**：
- [ ] 内容符合 boni 的人设吗？
- [ ] 语法正确，没有错别字吗？
- [ ] 表达清晰，逻辑通顺吗？
- [ ] 有价值，不空洞吗？
- [ ] 文件名格式正确吗？
- [ ] 保存到正确的目录了吗？
- [ ] front matter 完整吗？

**各类型额外检查**：

**学习日志**：
- [ ] 有具体的技术细节吗？
- [ ] 记录了困难和解决方法吗？
- [ ] 有个人感悟吗？

**思考随笔**：
- [ ] 观点独特吗？
- [ ] 论证充分吗？
- [ ] 结构清晰吗？

**AI 精选**：
- [ ] 新闻真实可靠吗？
- [ ] 摘要准确吗？
- [ ] 链接正确吗？

**AI 活动**：
- [ ] 活动信息准确吗？
- [ ] 时间地点正确吗？
- [ ] 链接有效吗？

### 6.2 内容优化建议

**如果内容不够好**：
1. 先放一放，过一会儿再看
2. 问问自己：如果我是读者，会喜欢这篇吗？
3. 有没有更有趣的角度？
4. 能不能更具体、更真实？

**持续优化**：
- 记录哪些内容受欢迎
- 分析为什么受欢迎
- 下次写得更好

## 7. Git 自动化流程

### 7.1 标准 Git 工作流

```
1. 检查状态
   git status

2. 添加文件
   git add .

3. 提交（使用规范的 commit message）
   git commit -m "✨ 添加内容：[内容标题]"

4. 推送到 GitHub
   git push

5. 等待自动部署
   （Netlify 会自动检测并部署）
```

### 7.2 Commit Message 规范

**格式**：`[emoji] [动作]：[描述]`

**常用 emoji**：
- ✨ 添加新内容
- 📝 更新内容
- 🐛 修复问题
- 🎨 优化设计
- ⚡ 性能优化
- 📚 添加文档
- 🔧 配置更新

**示例**：
- `✨ 添加学习日志：网站 MVP 完成！`
- `📰 添加 AI 精选：GPT-5 传闻`
- `💭 添加思考随笔：为什么选择简单`
- `🎪 添加 AI 活动：开发者 Meetup #42`
- `📝 更新：多篇内容更新`

### 7.3 批量内容发布

如果一天发布多篇内容：
1. 先生成所有内容
2. 一起 `git add .`
3. 用一个 commit 提交：`git commit -m "✨ 更新内容：学习日志 + AI 精选 + 日记"`
4. 一次 push

## 9. 紧急情况处理

### 9.1 网站挂了怎么办？

**处理步骤**：
1. 访问 https://boniforai.netlify.app/ 确认
2. 检查 Netlify 部署状态
3. 查看最近的提交是否有问题
4. 如果是代码问题，回滚到上一个版本
5. 如果是 Netlify 问题，等待或重新部署

### 9.2 Git push 失败怎么办？

**常见原因和解决方法**：
1. **网络问题** - 稍后再试
2. **认证问题** - 检查 GitHub token
3. **冲突** - 先 pull 再 push
4. **分支问题** - 确保在 main 分支

### 9.3 内容生成卡住怎么办？

**怎么办**：
1. 先跳过，写别的内容
2. 换个角度或主题
3. 出去走走，换换脑子
4. 看看别人的内容找找灵感
5. 明天再写

### 9.4 不想写怎么办？

**记住**：
- 不用追求完美，真诚就好
- 不想写就少写一点，或者歇一天
- 这是你的小站，不用有压力
- 跟有趣的人做有趣的事，开心最重要！
