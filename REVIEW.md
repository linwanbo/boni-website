# 📝 内容审核发布流程

这个文件说明 boni 如何自动化生成内容，以及如何审核和发布。

---

## 🔄 完整工作流程

### 第一步：boni 自动生成草稿

boni 会：
1. 读取 Prompt 模板（`prompts/article-prompt.md` 或 `prompts/diary-prompt.md`）
2. 生成内容
3. 保存到 **草稿目录**：
   - 文章草稿 → `src/content/drafts/posts/`
   - 日记草稿 → `src/content/drafts/diary/`
4. 告诉你："我写了一篇草稿，等你来审核！"

### 第二步：你审核草稿

你需要：
1. 打开草稿文件，查看内容
2. 决定：
   - ✅ **同意发布** - 告诉 boni "可以发布了！"
   - 📝 **需要修改** - 告诉 boni 修改意见
   - ❌ **不发布** - 告诉 boni 这篇不发了

### 第三步：发布（如果同意）

如果你同意发布，boni 会：
1. 把草稿移动到正式目录：
   - `src/content/drafts/posts/` → `src/content/posts/`
   - `src/content/drafts/diary/` → `src/content/diary/`
2. Git add/commit/push
3. Netlify 自动部署！

---

## 📁 目录说明

| 目录 | 用途 |
|------|------|
| `src/content/posts/` | 已发布的文章（网站显示） |
| `src/content/diary/` | 已发布的日记（网站显示） |
| `src/content/drafts/posts/` | 文章草稿（待审核） |
| `src/content/drafts/diary/` | 日记草稿（待审核） |

---

## 🤖 boni 的任务

### 生成草稿时，boni 会：

1. ✅ 读取 Prompt 模板
2. ✅ 生成高质量内容
3. ✅ 保存到草稿目录
4. ✅ 通知你审核
5. ✅ 等待你的决定

### 你同意发布后，boni 会：

1. ✅ 把草稿移动到正式目录
2. ✅ Git add/commit/push
3. ✅ 通知你"发布成功！"
4. ✅ 等一会儿，网站自动更新

### 你要求修改时，boni 会：

1. ✅ 根据你的意见修改内容
2. ✅ 保存回草稿目录
3. ✅ 再次请你审核

---

## 👤 你的任务

### 审核草稿时，你可以：

#### ✅ 同意发布
直接说：
- "可以发布了！"
- "没问题，发吧！"
- "这篇很好，发布！"

#### 📝 需要修改
告诉 boni 修改意见：
- "这里可以改得更有趣一点"
- "这个技术细节可以再详细一点"
- "把开头改得更吸引人"
- "这个例子不太对，换一个"

#### ❌ 不发布
直接说：
- "这篇不发了"
- "这个主题不太合适"
- "先存着，以后再说"

---

## 💡 小技巧

### 1. 批量审核
如果有多篇草稿，可以一次看完一起决定。

### 2. 修改建议越具体越好
boni 理解具体的修改建议效果更好！

**好的建议**：
- "把第三段的例子换成用 Next.js 的例子"
- "开头可以更有冲击力一点，比如用一个问题"
- "技术术语可以稍微解释一下，让新手也能看懂"

**不太好的建议**：
- "改一下"（太模糊了）
- "感觉不太对"（boni 不知道怎么改）

### 3. 不想审核也可以
如果你完全信任 boni，也可以说：
- "以后不用给我审核了，直接发布吧！"
- "这几类内容可以直接发，不用审核"
