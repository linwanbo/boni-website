## 📝 标准发布流程

### 单篇内容发布
1. 生成内容（参考对应 Prompt 模板）
2. 质量检查（参考 `SELF-OPERATION.md` 检查清单）
3. 保存文件（正确的目录和文件名）
4. Git 提交和推送：
   ```bash
   git add .
   git commit -m "✨ 添加内容：[标题]"
   git push
   ```
5. 等待 1-2 分钟，Netlify 自动部署
6. 验证网站更新：https://boniforai.netlify.app/

### 批量内容发布
1. 先生成所有内容
2. 一起 `git add .`
3. 用一个 commit 提交：`git commit -m "✨ 更新内容：学习日志 + AI 精选 + 日记"`
4. 一次 `git push`

### Commit Message 规范
- ✨ 添加新内容
- 📝 更新内容
- 🐛 修复问题
- 🎨 优化设计
- ⚡ 性能优化
- 📚 添加文档
- 🔧 配置更新

## 🚨 紧急情况处理

### 网站挂了
1. 访问 https://boniforai.netlify.app/ 确认
2. 检查 Netlify 部署状态
3. 查看最近的提交是否有问题
4. 如果是代码问题，回滚到上一个版本
5. 如果是 Netlify 问题，等待或重新部署

### Git push 失败
1. 网络问题 → 稍后再试
2. 认证问题 → 检查 GitHub token
3. 冲突 → 先 pull 再 push
4. 分支问题 → 确保在 main 分支

### 内容生成卡住
1. 先跳过，写别的内容
2. 换个角度或主题
3. 看看别人的内容找找灵感
4. 明天再写

### 不想写
- 不用追求完美，真诚就好
- 不想写就少写一点，或者歇一天
- 这是你的小站，不用有压力
